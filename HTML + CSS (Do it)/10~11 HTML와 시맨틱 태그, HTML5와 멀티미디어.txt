10-1 HTML5 문서

시맨틱 태그
= HTML5에서 태그 이름만 보고도 문서 구조에서 어떤 역할을 하는지 쉽게 이해할 수 있는 태그



10-2 문서 구조를 위한 HTML5 시맨틱 태그

<header> 태그
= 머리말 지정
- 주로 페이지의 맨 위쪽이나 왼쪽에 삽입
- 헤더의 내용으로는 주로 <form> 태그를 사용해 검색 창을 넣거나, <nav> 태그를 사용해 사이트 메뉴를 넣음
- 본문 중에 사용해 해당 부분의 머리말로 사용할 수도 있음

<nav> 태그
= 내비게이션 링크
- 같은 사이트 안의 문서나 타 사이트의 문서로 연결하는 링크 모음

<section> 태그
= 주제별 콘텐츠 영역
- 문서에서 주제별로 콘텐츠를 묶을 때 사용
- 섹션 제목을 나타내는 <hn> 태그가 함께 사용됨

<article> 태그
= 콘텐츠 내용
- 웹 상의 실제 내용
- 태그 적용 부분을 떼어내 독립적으로 배포하거나 재사용하더라도 완전히 하나의 콘텐츠

<aisde> 태그
= 본문 이외의 내용 (사이드바)

<iframe> 태그
= 웹 문서 안에 다른 외부 문서 삽입 (인라인 프레임)

<footer> 태그
= 웹 문서의 끝자락에 들어감
- 사이트 제작자의 연락처 정보와 저작권 정보를 표시

<address> 태그
= 제작자 정보와 연락처 정보
- 주로 <footer> 태그 안에 사용
- 단순 우편 주소는 <p> 태그 사용

<div> 태그
= 주로 콘텐츠를 묶어 시각적 효과를 적용할 때, 즉 콘텐츠에 CSS를 적용할 때 사용



10-3 IE8 이하 버전에서는 어떻게 하나요?

HTML5 Shiv 사용하기
- http://github.com/aFarkas/html5shiv 에서 [latest zip package]를 클릭해 다운로드
- 압축을 푼 후, src 폴더 안의 "html5shiv.js"파일을 원하는 폴더로 복사해 사용
- <head>와 </head>사이에 다음과 같은 소스를 추가하면 IE8 이하 버전에서도 시맨틱 태그를 인식할 수 있음
*	<!--[if lt IE 9]>
		<script src="html5shiv.js의 경로"></script>
	<![endif]-->



11-1 웹과 멀티미디어

플로그인 프로그램
- HTML4까지는 플러그인 프로그램을 연결해서 사용
- HTML5 웹 표준 이후 웹 브라우저에서 직접 멀티미디어 재생

HTML5와 비디오 코덱
- 인코딩(encoding) : 원본 비디오를 컴퓨터에서 사용할 수 있는 비디오 파일로 변환하는 과정
- 디코딩(decoding) : 컴퓨터 비디오 파일에 있는 비디오 정보를 가져와 플레이어에 보여주는 과정
- 비디오 코덱 : 인코딩과 디코딩을 수행
	- HTML5에서는 브라우저에서 직접 재생할 수 있는 비디오 코덱만 허용
	- 대부분의 브라우저에 H.264 코덱을 지원하므로, .mp4 파일을 기본적으로 사용
	- 무료이면서 최신 코덱은 v9 코덱을 이용한 .webm 파일도 함께 사용
	- .ogv 파일 : 화질은 별로지만 무료라는 장점 때문에 .webm 등장 이전에 많이 사용됨

HTML5와 오디오 코덱
- 대부분의 브라우저에서는 .mp3 사용



11-2 오디오 & 비디오 재생하기

<audio> 태그
= 오디오 파일 삽입
- 주로 .mp3 파일을 사용함
= <audio src="오디오 파일 경로"></audio>
	- width, height : 비디오 크기 조절
	- autoplay : 자동 재생 (크롬 브라우저에서는 자동 재생이 되지 않음)
	- controls : 웹 화면에 컨트롤 막대를 표시
	- loop : 반복 재생
	- muted : 오디오를 재생해 진행하지만 소리는 끔
	- preload : 재생 버튼을 눌러 재생하기 전에 오디오 파일을 다운로드해 준비해 둠
		- none : 미리 다운로드하지 않고, 사용자가 재생 버튼을 눌러야만 다운로드
		- metadata : 미디어 파일 전체를 다운로드하지 않고, 메타 정보만 다운로드
		- auto : [기본값] 사용자가 즉시 이용할 수 있도록 웹 로드 시 모두 다운로드
	- poster : 포스터 이미지(비디오를 재생할 수 없는 경우, 대신 표시하는 이미지) 지정
		- src 속성 사용

<video> 태그
= 비디오 파일 삽입
- <audio> 태그와 속성값이 같음

<source> 태그
= 여러 미디어 파일을 한꺼번에 지정하기
- 브라우저애 따라 지원하는 코덱이 다르기 때문에 한 가지 파일만 사용했을 경우 동작하지 않을 수 있음
	- <video>태그와 <source> 태그를 함께 사용해 여러 개의 파일을 지정해야 함
= <source src="비디오 파일 경로" type="video/확장자">
	- src : [필수] 미디어 파일의 경로를 지정
	- type : 미디어 파일의 유형을 알림
ex )	<video controls>
		<source src="Painting.mp4" type="video/mp4">
		<source src="Painting.webm" type="video/webm">
		<source src="Painting.ogv" type="video/ogv">
	</video>
- 여러 형식의 파일을 올렸는데도, 웹 서버에 업로드한 후 재생되지 않는 경우가 있음
	- 웹 서버에서 확장자를 인식하지 못했기 때문
	- 웹 서버 관리자에게 MIME 유형을 설정해 달라고 요청해야 함

<track> 태그
= 비디오에 외부 자막 파일을 연결하는 태그
= <track kind="자막 종류" src="경로" srclang="언어" label="제목" default>
	- kind : 자막 종류 지정
		- subtitles : [기본값] 자막
		- captions : 청각장애인용 자막
	- srclang : 사용한 언어 (ko, en, ja 등)
	- label : 자막이 여러 개일 경우, 자막을 식별할 수 있도록 제목을 달아줌
	- default : 자막 파일이 여러 개일 경우, 기본으로 사용할 자막을 지정

webVTT 자막 파일 (= 웹 비디오 텍스트 트랙(Web Video Text Track))
= 모든 브라우저에서 공식적으로 지원하는 자막 파일 형식
- .vtt 확장자 파일
- 메모장 같은 텍스트 편집기에 입력한 후 .vtt 파일로 저장해 사용
- 자막 내용과 시간 정보를 함께 담고 있음 (HH:MM:SS.ttt 형식)
ex )	00:00.000 --> 00:02.172
